# Data

This directory contains the dataset, model predictions, and evaluation outputs used in the paper.
Raw data and model outputs are not tracked in version control due to size.

## Download the Dataset

We use the **Next Day Wildfire Spread (NDWS)** dataset from [Huot et al. (2022)](https://arxiv.org/abs/2112.02922).

### Option 1: Kaggle CLI (recommended)

```bash
# Requires Kaggle credentials (~/.kaggle/kaggle.json)
# See: https://www.kaggle.com/docs/api#authentication
uv run fire-download-ndws --dataset fantineh/next-day-wildfire-spread --output-dir data/raw
```

### Option 2: Manual download

1. Visit [kaggle.com/datasets/fantineh/next-day-wildfire-spread](https://www.kaggle.com/datasets/fantineh/next-day-wildfire-spread)
2. Download and extract into `data/raw/`

The expected structure after download:

```
data/
├── raw/                          # ~3.7 GB, TFRecord files
│   ├── next_day_wildfire_spread_train_00.tfrecord
│   ├── ...
│   ├── next_day_wildfire_spread_eval_*.tfrecord
│   └── next_day_wildfire_spread_test_*.tfrecord
├── splits/                       # Tracked — deterministic split manifest
│   └── ndws_seed42_70_15_15.json
├── predictions/                  # Generated by training scripts
│   ├── tabular_baseline/
│   └── spatial_baseline/
├── evaluation/                   # Generated by evaluation scripts
└── README.md
```

## Reproducing from Scratch

```bash
# 1. Install dependencies
uv sync

# 2. Download dataset
uv run fire-download-ndws --dataset fantineh/next-day-wildfire-spread --output-dir data/raw

# 3. Prepare splits and verify loaders
uv run fire-prepare-phase1 \
  --data-root data/raw \
  --input-format auto \
  --split-manifest data/splits/ndws_seed42_70_15_15.json \
  --tabular-sample-rate 0.05

# 4. Train models (see root README.md for full commands)

# 5. Run CRC calibration and evaluation
uv run fire-calibrate-crc
uv run fire-evaluate
```
